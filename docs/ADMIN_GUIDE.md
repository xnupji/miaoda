# HTP挖矿平台 - 管理后台功能说明

## 概述

管理后台是HTP挖矿平台的核心管理工具，仅限管理员访问。提供用户管理、审核管理、开发者设置等全方位的平台管理功能。

## 访问权限

- **访问条件**：必须是管理员角色（role = 'admin'）
- **第一个用户**：平台第一个注册的用户自动成为管理员
- **权限提升**：管理员可以将其他用户提升为管理员

## 功能模块

### 1. 统计概览

在管理后台顶部显示关键统计数据：

- **总用户数**：平台注册用户总数
- **待审核提币**：等待审核的提币申请数量（黄色高亮）
- **待审核主节点**：等待审核的主节点申请数量（黄色高亮）
- **主节点数量**：当前主节点用户数量

### 2. 用户管理

#### 功能说明
管理所有注册用户的信息和权限。

#### 显示信息
- 用户名
- HTP余额
- USDT余额
- 邀请人数
- 钱包激活状态（已激活/未激活）
- 注册时间
- 用户角色（管理员/主节点/普通用户）

#### 操作功能
点击"编辑"按钮可以修改用户信息：

1. **修改余额**
   - HTP余额：可以增加或减少用户的HTP代币余额
   - USDT余额：可以增加或减少用户的USDT余额
   - 用途：奖励用户、处理纠纷、调整账户

2. **修改角色**
   - 普通用户：默认角色，基础权限
   - 主节点：拥有推广权限，可以获得激活奖励
   - 管理员：拥有完整的后台管理权限

3. **钱包激活状态**
   - 可以手动激活用户钱包
   - 激活后用户可以提取HTP代币
   - 用于处理特殊情况或奖励用户

#### 使用场景
- 用户支付了激活费用但系统未自动激活：手动激活钱包
- 奖励活跃用户：增加HTP或USDT余额
- 处理用户投诉：调整余额或权限
- 提升用户为主节点：修改角色为master_node
- 添加新管理员：修改角色为admin

### 3. 提币审核

#### 功能说明
审核用户的HTP代币提币申请。

#### 显示信息
- 用户名
- 提币数量（HTP）
- 目标地址（用户的钱包地址）
- 申请时间
- 审核状态（待审核/已通过/已拒绝）

#### 操作功能
1. **通过申请**
   - 确认用户的提币申请
   - 用户可以将HTP转出到钱包
   - 系统自动扣除用户的HTP余额

2. **拒绝申请**
   - 需要填写拒绝原因
   - 用户会收到拒绝通知
   - HTP余额不会扣除

#### 审核流程
1. 用户在钱包页面提交提币申请
2. 管理员在后台查看申请详情
3. 验证用户钱包是否已激活
4. 检查用户HTP余额是否充足
5. 确认目标地址是否正确
6. 通过或拒绝申请

#### 注意事项
- 只有钱包已激活的用户才能提币
- 提币数量不能超过用户的HTP余额
- 拒绝申请时必须填写原因
- 通过后无法撤销，请谨慎操作

### 4. 主节点审核

#### 功能说明
审核用户的主节点申请。

#### 显示信息
- 用户名
- 申请理由
- 当前激活进度（已激活地址数量）
- 申请时间
- 审核状态（待审核/已通过/已拒绝）

#### 操作功能
1. **通过申请**
   - 直接将用户角色提升为主节点
   - 用户获得推广权限
   - 可以获得激活奖励（每个地址3 USDT）

2. **拒绝申请**
   - 需要填写拒绝原因
   - 用户保持普通用户角色

#### 审核标准
- 用户是否有推广能力
- 申请理由是否充分
- 用户活跃度和信誉
- 平台主节点配额

#### 主节点权益
- 完成10万个钱包地址激活
- 每激活1个地址奖励3 USDT
- 专属推广工具和邀请码
- 优先获得平台更新信息

### 5. 开发者设置

#### 功能说明
管理开发者USDT接收地址，用于接收用户支付的钱包激活费用。

#### 显示信息
- **当前开发者地址**：显示正在使用的BSC钱包地址
- **新的开发者地址**：输入框，用于更新地址

#### 操作功能
1. **查看当前地址**
   - 显示完整的钱包地址
   - 可以复制地址用于验证

2. **更新地址**
   - 输入新的BSC钱包地址
   - 点击"更新开发者地址"按钮
   - 系统验证地址格式
   - 更新成功后立即生效

#### 重要说明
- **仅管理员可修改**：此功能只有管理员角色可以访问
- **BSC网络**：地址必须是BSC（币安智能链）网络的钱包地址
- **立即生效**：更新后，所有新的激活支付都会使用新地址
- **历史记录**：已支付的激活费用不受影响
- **安全性**：请确保地址正确，避免资金损失

#### 使用场景
- 更换开发者钱包
- 分散资金管理
- 安全性升级
- 多签钱包切换

#### 验证方法
更新地址后，可以通过以下方式验证：
1. 在钱包页面查看激活界面，确认显示的是新地址
2. 访问 [BSCScan](https://bscscan.com/) 搜索新地址
3. 让测试用户尝试激活钱包，确认支付到新地址

## 安全建议

### 权限管理
1. **谨慎授予管理员权限**：只将信任的用户提升为管理员
2. **定期审查管理员列表**：确保所有管理员都是活跃且可信的
3. **记录操作日志**：重要操作应该有记录和追溯

### 审核流程
1. **仔细核对信息**：审核前确认所有信息准确无误
2. **验证链上记录**：对于激活支付，应该在BSCScan上验证
3. **及时处理申请**：避免用户等待时间过长
4. **保持沟通**：拒绝申请时应该给出明确原因

### 地址管理
1. **备份私钥**：确保开发者钱包的私钥安全备份
2. **使用硬件钱包**：建议使用硬件钱包存储大额资金
3. **定期检查余额**：监控开发者地址的USDT余额
4. **多签钱包**：考虑使用多签钱包提高安全性

## 常见问题

### Q1: 如何添加新的管理员？
A: 在用户管理标签页，找到目标用户，点击"编辑"，将角色改为"管理员"，保存即可。

### Q2: 用户支付了激活费用但钱包未激活怎么办？
A: 
1. 在BSCScan上验证用户的转账记录
2. 确认转账金额和目标地址正确
3. 在用户管理中找到该用户，点击"编辑"
4. 勾选"钱包已激活"，保存

### Q3: 如何处理提币申请？
A:
1. 进入"提币审核"标签页
2. 查看待审核的申请
3. 验证用户钱包已激活
4. 确认HTP余额充足
5. 点击"通过"或"拒绝"

### Q4: 主节点申请的审核标准是什么？
A:
- 用户活跃度高
- 有推广能力和资源
- 申请理由充分
- 平台需要更多主节点

### Q5: 更新开发者地址会影响已支付的用户吗？
A: 不会。已经支付的激活费用不受影响，只有新的支付会使用新地址。

### Q6: 如何验证用户的激活支付？
A:
1. 复制开发者地址
2. 访问 https://bscscan.com/
3. 搜索开发者地址
4. 查看"Transactions"标签
5. 找到用户的转账记录
6. 确认金额和发送地址

### Q7: 可以批量操作吗？
A: 目前不支持批量操作，每个用户需要单独编辑。未来版本可能会添加批量功能。

### Q8: 如何撤销已通过的提币申请？
A: 已通过的申请无法撤销。如果需要调整，可以在用户管理中手动修改用户的HTP余额。
